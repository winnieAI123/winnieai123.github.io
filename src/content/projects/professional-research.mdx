## 覆盖的 9 类研究

| 研究类型 | 一句话说明 |
| --- | --- |
| 公司研究 | 单一公司深度起底：业务、财务、竞争力 |
| 行业全景 | 赛道格局、市场规模与玩家地图 |
| 产品研究 | 单品 / 竞品的功能与定位拆解 |
| 趋势研判 | 方向、拐点与情景预测 |
| 政策追踪 | 政策动向及其对行业的影响 |
| 榜单分析 | 排行榜 / 评测数据横向对比 |
| 学术简报 | 论文速读与核心要点提炼 |
| KOL 周报 | 行业大佬观点聚合与梳理 |
| 热点速评 | 突发事件的多方观点快评 |

## 为什么做这个

这个 Skill 不是为我一个人做的。我先把小组里**60+ 个高频研究需求**收集起来，发现它们其实反复落在少数几种模式上，于是把它们抽象、归并成 **9 大类标准研究框架**，再为每一类配好模板与采集策略，做成一个可一键调用的引擎。

研究本身是非标的，但**研究的"方法"可以标准化**。沉淀成 Skill 后，团队里任何人接到一个新课题，都能直接复用——把原本各做各的、质量看心情的研究，变成了**全组共享、有稳定下限的研究资产**。

## 设计理念

这个 Skill 的核心价值不是"能做研究"，而是**让研究质量有稳定的下限**。模板驱动意味着哪怕是不熟悉的主题，产出也不会漏掉关键维度；假设驱动意味着带着问题去采集，而不是漫无目的堆资料；多源采集意味着结论不建立在单一信息源上。把一次性的努力，沉淀成可复用的资产。
